# Questions

The open research agenda and ignorance ledger — what the oracle knows it does not yet know.

## What belongs here

One note per open question. Use `type: question`. A good question records *why it matters* (decision relevance), the *answer standard* (what evidence would settle it), candidate sources, and links to the contradictions, models, and findings it touches. Tracking ignorance explicitly is what keeps the oracle honest about the edges of its knowledge instead of confabulating.

## Mutability

Mutable hub. Update status as investigation proceeds (e.g. `open`, `investigating`, `answered`, `parked`). When a question is answered, link the resolving `Finding` or `Source` and set status accordingly — do not delete it; an answered question is a record of how a belief was earned.

## Sensitivity

The *fact that a question is open* can itself be sensitive (it can reveal strategic uncertainty). Default to `internal`; raise to `confidential` when the question exposes a vulnerability or unannounced direction. No secrets in the body.

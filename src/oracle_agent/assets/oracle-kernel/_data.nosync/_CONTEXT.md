# _data.nosync

Raw, snapshotted, mirrored, derived, and warehouse data lives here.

## Organization

- Prefer connector-specific raw stores (one folder per connector) plus a derived warehouse when cross-source analysis becomes recurring.
- `index/` holds the rebuildable retrieval index (FTS5 or pure-python fallback) built by `knowledge_index.py`. It is derived and can be regenerated; see `index/_CONTEXT.md`.
- `derived/` holds optional MemPalace and Graphify export corpora generated by `derived_memory.py`. These exports are rebuilt from indexed chunks and are not canonical records.

## Authority

Raw data is **not** the source of truth for every domain. Source authority is declared in `TRUTH-MAP.md` and resolved by `truth_map.py` before any material answer — a snapshot in `_data.nosync/` is evidence, not automatic truth. Locality (`external_only` / `snapshot_local` / `mirror_local`) records *where bytes physically live*; capture_tier (`manifest_only` / `snapshot` / `mirror`) records *how much we copy*. Both come from each connector's manifest.

Optional derived-memory engines follow the same authority rule. MemPalace can improve recall and Graphify can expose graph structure, but material answers still route through Source records, truth-map authority, and the answer protocol.

## Sensitivity

Raw stores often carry the most sensitive material in the oracle. Classify at ingest time (`intake_classify.py`), keep secrets in env only (never in data files), and respect the processing matrix before any external handling.
